Clash for Windows mixin配置备忘

Hui
2020-11-14 (Updated: 2020-11-14)
life, 网络

不在GFW环境下使用Clash for Windows还是会忽略掉很多错误的配置。由于我最近几天发现在丰富了CFW的mixin配置之后，主要是DNS的配置，TAP模式无法正常运行。

mixin的功能

​ 在cfw0.9.5版本更新后，支持向所有配置文件中注入公共属性设置。

​ 例如：在配置文件中统一添加 dns 字段，在启动或切换配置时，字段内容将会替换到原有配置文件中进行覆盖。

在有了这么强大的mixin功能后，我们通过注入配置即可设置很多通用的属性到profile中去。由于网络环境的特殊性，单是 dns 字段就包含非常多的定义项。跟随着clash内核的不断升级，参考了 wiki 中的说明，才逐渐明白如何调整这个属性。

DNS Example

dns:
  enable: true                   # set true to enable dns (default is false)  
  ipv6: false                    # default is false  
  listen: 0.0.0.0:53  
  default-nameserver:            # resolve dns nameserver host, should fill pure IP
                                 # aka Bootstrap DNS
    - 114.114.114.114      
    - 8.8.8.8      
  enhanced-mode: redir-host      # or fake-ip  
  fake-ip-range: 198.18.0.1/16   # if you don't know what it is, don't change it  
  use-hosts: true                # lookup hosts and return IP record  
  fake-ip-filter:                # fake ip white domain list, aka Always Real IP  
    - '*.lan'    
    - localhost.ptlogin2.qq.com    
  nameserver:
    - 114.114.114.114    
    - tls://dns.rubyfish.cn:853  # dns over tls    
    - https://1.1.1.1/dns-query  # dns over https    
  fallback:                      # concurrent request with nameserver, 
                                 # fallback used when GEOIP country isn't CN
    - tcp://1.1.1.1    
  fallback-filter:  
    geoip: true                  # default    
    ipcidr:                      # ips in these subnets will be considered polluted
      - 240.0.0.0/4

default-nameserver(Optional)

用于解析nameserver段内的DoH/DoT的ip地址的DNS，只接受IP格式的DNS。

enhanced-mode

DNS服务器内部有两种工作模式：redir-host和fake-ip模式。

redir-host模式

当客户端发起DNS请求时，Clash会将需要解析的域名发送到每个上游的DNS服务器，并将第一个返回最快的结果发送给客户端，类似于dnsmasq的–all-servers模式。然后Clash会等待客户端建立连接。

fake-ip模式

当客户端发起DNS请求时，Clash会直接返回一个虚假IP给客户端。同时，Clash会将需要解析的域名发送到各个上游的DNS服务器。最后，返回速度最快的第一个结果与假IP相关联。
一旦DNS有效期到期，Clash仍然会返回相同的假IP给客户端。同时，Clash会再次向上游请求解析结果，重新建立对应关系。

fake-ip-filter (Only available in fake-ip mode)

作用：始终将真实IP返回此列表中的域，以提高兼容性。支持通配符。主要是处理fake-ip模式下，微软等服务的稳定性。

fake-ip-filter:                
    - '*.lan'    
    - localhost.ptlogin2.qq.com    
    - '*.*.*.*.srv.nintendo.net'    
    - '*.*.*.stun.playstation.net'    
    - 'xbox.*.microsoft.com'    
    - '*.*.*.xboxlive.com'

fallback

定义多个上游DNS回退服务器，支持用udp、tcp、udp在特定的端口、tcp在特定的端口、DNS over TLS、DNS over HTTPS解析。
与命名服务器并发请求，当GEOIP国家不是CN时，使用fallback。

fallback:
  - 114.114.114.114
  - 87.118.100.175:110
  - tcp://1.1.1.1
  - tls://dns.rubyfish.cn:853 # dns over tls
  - https://1.1.1.1/dns-query # dns over https

YAML语法

cfw的mixin支持YAML和JavaScript两种语法，默认使用YAML。

YAML 是专门用来写配置文件的语言，非常简洁和强大，远比 JSON 格式方便。

它的基本语法规则如下。

  • 大小写敏感
  • 使用缩进表示层级关系
  • 缩进时不允许使用Tab键，只允许使用空格。
  • 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可

我踩坑的地方

  1. 使用缩进表示层级关系，相同层级的值一定要对齐。
  2. 缩进千万不要用Tab，否则只能清空格式重新排版。

我的配置

  mixin: # object
    dns:
     enable: true
     ipv6: false
     enhanced-mode: fake-ip # 或  redir-host
     listen: 0.0.0.0:53
     default-nameserver: # resolve dns nameserver host, should fill pure IP
      - 114.114.114.114
      - 8.8.8.8
     nameserver:
      - 119.29.29.29
      - tls://13800000000.rubyfish.cn:853
      - tcp://1.1.1.1
     fallback:
      - tls://dns.rubyfish.cn:853
      - https://1.1.1.1/dns-query
      - tls://1.0.0.1:853
      - tls://dns.google:853
     fake-ip-filter:
      - '*.lan'
      - 'dns.msftncsi.com'
      - 'www.msftncsi.com'
      - 'www.msftconnecttest.com'      
      - localhost.ptlogin2.qq.com
     fallback-filter: 
      geoip: true
      ipcidr:
        - 240.0.0.0/4
  interface-name: WLAN
