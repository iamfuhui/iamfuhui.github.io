    缘起：在我准备回国的时候，想着把这一次入境隔离的日常做一个记录。当时考虑只是拍一些照片，记录一下流程发布在社交媒体上面。后来觉得社交媒体碎片化太严重，还是需要像Blog这样方便阅读、更新和检索的平台更加合适一些。
    选择Hexo：在选择Blog的平台的时候，发现Blog服务商们好像都默默无闻，页面也会被插入很多广告。而基于Github.io的Blog程序，已经风行了好几年。今年年初的时候，我也尝试过学习如何搭建Hexo的Blog，当时遇到了一些问题，于是被我暂时搁置了。由于这次隔离的时间比较充裕，我就从头开始研究了Hexo的用法。加上之前也通过Typora这个软件接触到了md格式的文档写法。而Hexo的本地写作再同步到Github的流程也很方便。

    进入正题

    通过Hexo+GitHub Pages，生成静态的网页。Hexo是一款快速、简洁且高效的博客框架，可以将我们使用Markdown编辑的md格式的文件生成html资源文件。

    1. 注册Github

      这个过程就不展开了，选择自己喜欢的username，因为这个关系到你的Blog域名。

      开启Github Pages，只需要新建一个以 <username>.github.io 为名的repository，private亦可。

      选择主题后，可以很快捷地生成一个pages页面。

    2. 使用Hexo

      本文操作系统为64位Windows 10。

      安装Git Bash

      Git Bash，对于Windows的支持（如文件名、同步锁等）很友好。安装完成后左下角搜索即可打开Git Bash。MacOS上直接 brew install git 即可。

      $ git --version
git version 2.21.0.windows.1

      安装Node.js

      最新版的Node.js请从官网下载。我使用的是原博主提供的v9.11.1版本64位msi包。安装过程简明。MacOS上直接brew install node即可。

      $ node -v
v9.11.1

$ npm -v
5.6.0

      安装Hexo

      具体请依照官网，完成前面的步骤后应该只需要npm install -g hexo-cli。会有一些报错，但可以无视。

    遇到的问题

    创建新建一个以 <username>.github.io 为名的repository时，默认的branches名称为main。在设置Hexo的git同步时，某些教程写为master，导致同步错误。保持名称与github一致即可。
